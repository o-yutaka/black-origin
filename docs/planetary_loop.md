## Evolution protocol

The `system_evolution` stage runs a concrete protocol that:

- detects bottlenecks (stream and graph pressure),
- expands agents and pipelines,
- refactors subsystems as needed,
- coordinates with autogeneration and cluster scaling,
- feeds observability and security outputs back into the next cycle.
